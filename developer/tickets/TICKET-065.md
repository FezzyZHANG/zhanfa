# TICKET-065: 引入腾讯行情 Provider 并切换日线主源

- **优先级:** P1 - 高
- **状态:** 📋 待开始
- **依赖:** 043, 064
- **预计工时:** 3-5d

## 背景

东方财富 `stock_zh_a_hist` 在当前网络环境出现 `RemoteDisconnected`，现有未提交改动已经尝试用 AKShare 的 `stock_zh_a_hist_tx` 作为异常 fallback。调研确认腾讯可以覆盖沪深 A 股和指数日线，但直接把该包装器接进 `Fetcher.daily()` 存在字段契约、北交所、请求规模、复权一致性和授权风险。

完整调研见 [腾讯行情数据源迁移可行性报告](../reports/tencent-data-source-feasibility.md)。

## 目标

把腾讯建设为**个股日线和指数日线的可配置主 Provider**，保留按能力路由和一键回退。此次不是“删除 akshare”或“全部数据切腾讯”：分钟线、股票列表、指数/行业成分和财务数据继续使用现有来源。

## Critical 范围

- [ ] 记录项目用途、腾讯授权/协议评估和风险接受；未满足使用边界时不得在商业或对外生产环境默认启用直连接口。
- [ ] 为个股/指数日线抽出最小 Provider 边界，避免来源判断散落在 `Fetcher`、service 和 workflow。
- [ ] 实现腾讯日线 Provider，显式设置连接/读取超时，校验 HTTP 状态、业务响应和证券代码键。
- [ ] 正确映射上交所、深交所、北交所代码；至少覆盖 `600519`、`688001`、`000001`、`300750`、`920002` 和 `000300`。
- [ ] 统一输出 `open/high/low/close/volume/amount/turnover`；成交量按“手”、成交额按“万元”、换手率按“百分数”的来源单位显式转换并测试。
- [ ] 兼容腾讯 `day` / `qfqday` / `hfqday` 响应；不得依赖当前 AKShare 北交所起始日期探测的错误 fallback。
- [ ] 设计缓存水位增量更新，避免 TTL 过期后逐股重拉 2010 年至今；前复权数据必须有定期全量校准或除权事件重算策略。
- [ ] 加入限并发、指数退避 + 抖动、最大重试、熔断和分批断点续传；不得无上限并发抓取全市场。
- [ ] 提供明确配置，例如 `ZHANFA_DAILY_PROVIDER=tencent|akshare` 和独立 fallback 开关；关闭腾讯后无需改代码即可恢复现状。
- [ ] 日志和数据状态至少暴露 Provider、请求耗时、请求次数、重试、fallback 和最终失败原因。

## High 范围

- [ ] 对不少于 50 只代表性证券和 5 个指数做来源对账，覆盖沪主板、深主板、创业板、科创板、北交所、ST、停牌和上市不满一年标的。
- [ ] 不复权数据先验证交易日集合、OHLC、成交量、成交额和换手率；复权数据单独记录来源差异，禁止把不同来源片段静默拼接。
- [ ] 为初次全量回填与日常增量刷新分别建立耗时、请求量和失败率基线。
- [ ] 为响应缺字段、空数据、HTML/非 JSON、超时、429/5xx、无效证券和单只失败补齐单元测试。
- [ ] 保持 Store、Pipeline、策略、回测和现有 API 的统一 DataFrame 契约不变。
- [ ] 更新 `docs/data.md`（当前实际数据流）以及 `developer/architecture.md`、`developer/debug-notes.md`（Provider 设计、切换与排障）。

## 非目标

- 不在本工单内迁移 `stock_list()`、`index_components()`、`industry_stocks()`、`financial()`。
- 不用腾讯当日分笔接口替换 15/30/60 分钟线。
- 不新增对外实时行情 API；`qt.gtimg.cn` 批量快照能力另行建票。
- 不删除 akshare 依赖，也不移除现有回退路径。

## 验收标准

- [ ] 腾讯作为日线主 Provider 时，代表性沪深北股票和指数均可刷新并写入现有 parquet 目录。
- [ ] 返回列、单位、索引、排序和空值行为满足现有 Fetcher/Store/Pipeline 契约。
- [ ] 北交所 `920002` 不再被错误映射为深交所，且不会触发 AKShare `KeyError: 'day'`。
- [ ] 全市场日常刷新只请求缓存水位后的必要窗口；请求量可观测且有硬上限。
- [ ] 腾讯失败时按配置回退；两个 Provider 都失败时保留明确错误，不写空缓存或部分损坏缓存。
- [ ] 数据对账报告记录样本、日期范围、字段容差、差异和最终切换决策。
- [ ] `uv run ruff check src/` 通过。
- [ ] `uv run mypy src/` 通过。
- [ ] `uv run pytest -v` 通过。
- [ ] 相关 `docs/`、`developer/` 文档与实际实现一致。
