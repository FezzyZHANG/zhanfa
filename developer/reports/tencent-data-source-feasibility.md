# 腾讯行情数据源迁移可行性报告

> 评估日期：2026-07-14
>
> 评估范围：zhanfa 当前 `Fetcher` 所需的 A 股日线、指数日线、分钟线、股票列表、指数/行业成分和财务数据

## 结论

**部分可行。建议把腾讯切为“个股日线 + 指数日线”的候选主源，不建议把全部 akshare 能力一次性切到腾讯。**

腾讯财经 Web 接口目前可以稳定返回沪深 A 股和指数日线，底层原始响应还包含成交量、成交额和换手率；但本次没有找到腾讯面向第三方发布的正式财经数据 API 文档、鉴权方式、配额或 SLA。腾讯也没有为 zhanfa 当前所需的全 A 股列表、指数成分、行业成分、财务数据和 15/30/60 分钟线提供一组已文档化、可等价替换的接口。因此迁移应采用**按能力路由的混合数据源架构**，而不是把 `Fetcher` 全局改成单一腾讯源。

在生产环境默认启用直连腾讯前，还必须确认授权和使用边界。腾讯服务协议将服务限定为个人非商业使用（另有约定除外），并明确限制未经授权通过第三方工具获取服务中的数据；这一点是上线门槛，不是普通技术风险。

## 策略研究时间范围

本仓库的策略研究默认关注 **2018-01-01 及以后**的数据。腾讯 Provider 的首次日线回填、默认查询和数据对账均应以 `20180101` 为默认起点，不再沿用当前代码中的 `20100101`。

这是一项默认研究边界，而不是强制删除或禁止访问更早数据：已有的 2018 年以前缓存可以保留，调用方确有需要时也可以显式指定更早日期；但初始化、调度刷新和常规策略研究不得自动拉取 2018 年以前的数据。缓存命中时也必须按请求的起止日期切片，不能因为本地 parquet 含有旧数据而忽略研究窗口。

## 已验证的接口

### 1. 历史日线

- AKShare 包装：[`stock_zh_a_hist_tx`](https://akshare.akfamily.xyz/data/stock/stock.html#id44)
- 腾讯底层地址：`https://proxy.finance.qq.com/ifzqgtimg/appstock/app/newfqkline/get`
- 起始日期探测：`https://web.ifzq.gtimg.cn/other/klineweb/klineWeb/weekTrends`
- 输入：带市场前缀的证券代码、起止日期、`qfq` / `hfq` / 不复权
- 覆盖：个股日线和指数日线；前复权历史值会随除权除息发生变化

AKShare 1.18.64 的官方文档将返回列写为 `date/open/close/high/low/amount`，但同时注明最后一列单位为“手”。因此该列实际是**成交量**，不是 zhanfa 契约中的成交额。AKShare 包装器还会截断腾讯原始行，只保留前 6 列。

2026-07-14 直接请求腾讯底层接口时，原始日线行的形态为：

```text
[date, open, close, high, low, volume_lot, {}, turnover_pct, amount_10k_cny, ...]
```

其中 `turnover_pct` 和 `amount_10k_cny` 是根据腾讯同一响应中的实时行情字段和量价关系推断的语义，腾讯没有提供可引用的正式字段文档，实施时必须用契约测试继续校验，不能只依赖字段位置。

### 2. 批量实时行情

- 地址：`https://qt.gtimg.cn/q=sh600519,sz000001,bj920002,sh000300`
- 返回：GBK 编码、`~` 分隔的自定义文本
- 2026-07-14 实测：4 个标的一次请求约 0.50 秒；沪深股票和指数返回 88 个字段，北交所样本返回 87 个字段
- 能力：最新价、昨收、今开、成交量、成交额、换手率、PE/PB 等

该接口可作为未来实时快照能力，但不属于本次“历史日线切换”的必要范围。由于字段无正式版本契约，解析器必须按最小长度和证券类型做防御性校验。

### 3. 当日分笔

- AKShare 包装：`stock_zh_a_tick_tx_js`
- 腾讯底层地址：`http://stock.gtimg.cn/data/index.php`

该接口只覆盖当日分笔，AKShare 当前实现使用 HTTP 且通过 `eval` 解析响应，不适合作为 zhanfa 分钟线替代方案。本次迁移不接入。

## 能力矩阵

| zhanfa 能力 | 当前来源 | 腾讯可替代性 | 迁移判断 |
|---|---|---|---|
| `daily()` | 东方财富 `stock_zh_a_hist` | 可替代，但 AKShare 包装丢失成交额/换手率 | **本工单目标**；优先实现受控 Provider |
| `daily_batch()` | 逐股调用 `daily()` | 腾讯历史接口仍需逐股、逐年请求 | 可复用，但必须限速、重试和增量更新 |
| `index_daily()` | 新浪指数日线 | `stock_zh_index_daily_tx` 可覆盖 | **本工单目标**；需交叉对账 |
| `minute(15/30/60)` | 新浪 `stock_zh_a_minute` | 未发现已文档化的等价腾讯接口 | 保留现状 |
| `stock_list()` | akshare/东方财富 | 未发现等价腾讯公开接口 | 保留现状 |
| `index_components()` | 中证指数 | 未发现等价腾讯公开接口 | 保留现状 |
| `industry_stocks()` | 东方财富行业板块 | 未发现等价腾讯公开接口 | 保留现状 |
| `financial()` | 同花顺财务摘要 | 未发现等价腾讯公开接口 | 保留现状 |
| 实时快照 | 当前无独立 Provider | `qt.gtimg.cn` 可批量获取 | 后续独立工单，不阻塞日线迁移 |

## 本地验证结果

测试窗口为 2026-07-01 至 2026-07-13，前复权：

| 标的 | 结果 | 行数 | 单次耗时 | 备注 |
|---|---:|---:|---:|---|
| `sz000001` 平安银行 | 成功 | 9 | 1.87s | 6 列 |
| `sh600519` 贵州茅台 | 成功 | 9 | 2.00s | 6 列 |
| `sh688001` 华兴源创 | 成功 | 9 | 1.50s | 科创板 |
| `sz300750` 宁德时代 | 成功 | 9 | 1.62s | 创业板 |
| `bj920002` 万达轴承 | AKShare 包装失败 | - | 1.75s | `KeyError: 'day'` |
| `sh000300` 沪深 300 | 成功 | 9 | 0.77s | 指数日线 |

北交所失败不是腾讯底层不支持。直接请求 `newfqkline/get` 并使用 `bj920002` 可以得到 9 行 `qfqday`；失败发生在 AKShare 的起始日期探测 fallback 固定读取 `day`。同时，zhanfa 当前“6 开头走上交所，否则走深交所”的临时代码会把 `920xxx` 错误映射成 `sz920xxx`。因此北交所代码映射和包装器缺陷必须在切换前解决。

## 性能与限流风险

AKShare 的腾讯包装器先发一次请求探测最早日期，再按年份循环请求历史数据。按本仓库确定的默认起点 2018 年、约 5,200 只 A 股估算，一次全市场完整回填约需要：

```text
5,200 × (1 次起始日期探测 + 9 个年度请求) ≈ 52,000 次上游请求
```

这与当前 6 小时 TTL 过期后重新拉完整历史的行为不兼容，容易造成耗时过长、被限流或封禁。切换工单必须同时完成：

1. 基于缓存水位的增量更新，而不是每次重拉 2018 年至今；
2. 前复权数据的定期全量校准或除权事件重算，避免历史价格漂移；
3. 有上限的并发、指数退避、抖动、超时和熔断；
4. 首次全市场回填分批执行，并可断点续传；
5. 记录来源、请求数、延迟、失败类型和 fallback 次数。

## 数据契约风险

1. **字段错位**：AKShare 腾讯包装返回的 `amount` 实为成交量（手）；若直接交给当前 `_clean_ohlcv()`，会缺少 `volume`，破坏下游契约。
2. **单位差异**：原始成交额推断为万元，需要转换为元；换手率推断为百分数，需要与当前东方财富口径保持一致。
3. **复权差异**：不同数据源的前/后复权算法可能不同，迁移前不能把两个来源的数据片段无条件拼接。
4. **北交所映射**：当前 `920xxx` 必须使用 `bj` 前缀；不能仅凭“是否 6 开头”判断市场。
5. **响应形态不稳定**：个股可能返回 `day`、`qfqday` 或 `hfqday`；实时行情不同市场的字段数也不完全相同。
6. **来源不可追踪**：现有 parquet 没有 Provider 元数据，混用来源后难以定位差异；至少应在刷新日志和状态 API 中记录来源。

## 合规与稳定性门槛

- 本次检索未找到腾讯面向第三方发布的正式财经数据 API、配额、版本策略或 SLA。
- [腾讯服务协议](https://edu.tencent.com/agreement.html)第 4.1 条限定个人非商业使用（另有约定除外），第 4.5 条限制未经授权通过第三方工具获取服务数据。
- [AKShare 项目说明](https://github.com/akfamily/akshare#disclaimer)声明数据仅供学术研究、接口可能因不可控因素被移除。

因此默认切换前应由项目所有者记录用途和风险接受；若未来用于对外服务、商业用途或大规模生产抓取，应先取得数据授权或改用有正式许可/SLA 的供应商。

## 推荐迁移方案

```text
Fetcher / Workflows
        |
        +-- DailyDataProvider
              +-- TencentDailyProvider  (个股/指数日线候选主源)
              +-- AkshareDailyProvider  (可配置回退)

其他能力仍按现状路由：
minute -> Sina
stock_list / industry -> 现有 akshare 来源
index_components -> 中证指数
financial -> 同花顺
```

不要把腾讯分支继续堆进现有 `_clean_ohlcv()`；Provider 应先把来源特有字段和单位转换成统一的 `date/open/high/low/close/volume/amount/turnover` 契约，再交给 Store/Pipeline。

实施范围和验收标准见 [TICKET-065](../tickets/TICKET-065.md)。
