# TICKET-065: 引入腾讯行情 Provider 并切换日线主源

- **优先级:** P1 - 高
- **状态:** ✅ 已完成
- **依赖:** 043, 064
- **预计工时:** 3-5d

## 背景

东方财富 `stock_zh_a_hist` 在当前网络环境出现 `RemoteDisconnected`，现有未提交改动已经尝试用 AKShare 的 `stock_zh_a_hist_tx` 作为异常 fallback。调研确认腾讯可以覆盖沪深 A 股和指数日线，但直接把该包装器接进 `Fetcher.daily()` 存在字段契约、北交所、请求规模、复权一致性和授权风险。

完整调研见 [腾讯行情数据源迁移可行性报告](../reports/tencent-data-source-feasibility.md)。

## 目标

把腾讯建设为**个股日线和指数日线的可配置主 Provider**，保留按能力路由和一键回退。此次不是“删除 akshare”或“全部数据切腾讯”：分钟线、股票列表、指数/行业成分和财务数据继续使用现有来源。

策略研究和日线初次回填默认从 **2018-01-01** 开始。更早数据只在调用方显式指定时按需获取，不属于初始化、定时刷新或常规研究的默认范围。

## Critical 范围

- [x] 记录项目用途、腾讯授权/协议评估和风险接受；未满足使用边界时不得在商业或对外生产环境默认启用直连接口。
- [x] 为个股/指数日线抽出最小 Provider 边界，避免来源判断散落在 `Fetcher`、service 和 workflow。
- [x] 实现腾讯日线 Provider，显式设置连接/读取超时，校验 HTTP 状态、业务响应和证券代码键。
- [x] 正确映射上交所、深交所、北交所代码；至少覆盖 `600519`、`688001`、`000001`、`300750`、`920002` 和 `000300`。
- [x] 统一输出 `open/high/low/close/volume/amount/turnover`；成交量按“手”、成交额按“万元”、换手率按“百分数”的来源单位显式转换并测试。
- [x] 兼容腾讯 `day` / `qfqday` / `hfqday` 响应；不得依赖当前 AKShare 北交所起始日期探测的错误 fallback。
- [x] 将 Fetcher、stock service 和股票 API 的默认日线起点从 `20100101` 统一为 `20180101`，并集中配置，避免多处硬编码漂移。
- [x] 缓存命中和网络返回都必须遵守请求的 `start` / `end` 范围；已有 2018 年以前缓存可以保留，但默认调用不得把更早数据返回给策略研究。
- [x] 设计缓存水位增量更新，避免 TTL 过期后逐股重拉 2018 年至今；前复权数据必须有定期全量校准或除权事件重算策略。
- [x] 加入进程内共享 QPS 限速、限并发、指数退避 + 抖动、最大重试、熔断和分批断点续传；重试计入限速额度，不得无上限抓取全市场。
- [x] 提供明确配置，例如 `ZHANFA_DAILY_PROVIDER=tencent|akshare` 和独立 fallback 开关；关闭腾讯后无需改代码即可恢复现状。
- [x] 日志和数据状态至少暴露 Provider、请求耗时、请求次数、重试、fallback 和最终失败原因。

## High 范围

- [x] 不复权数据先验证交易日集合、OHLC、成交量、成交额和换手率；复权数据单独记录来源差异，禁止把不同来源片段静默拼接。
- [x] 为初次全量回填与日常增量刷新分别建立耗时、请求量和失败率基线。
- [x] 为响应缺字段、空数据、HTML/非 JSON、超时、429/5xx、无效证券和单只失败补齐单元测试。
- [x] 保持 Store、Pipeline、策略、回测和现有 API 的统一 DataFrame 契约不变。
- [x] 更新 `docs/data.md`（当前实际数据流）以及 `developer/architecture.md`、`developer/debug-notes.md`（Provider 设计、切换与排障）。

## 非目标

- 不在本工单内迁移 `stock_list()`、`index_components()`、`industry_stocks()`、`financial()`。
- 不用腾讯当日分笔接口替换 15/30/60 分钟线。
- 不新增对外实时行情 API；`qt.gtimg.cn` 批量快照能力另行建票。
- 不删除 akshare 依赖，也不移除现有回退路径。

## 交付记录

2026-07-14 完成。临时 AKShare 腾讯 fallback 已收敛为独立 `DailyProvider`，腾讯成为本地非商业研究模式主源，akshare 保留为可配置主源/回退。实现包含沪深北与指数映射、原始字段和单位契约、超时/重试/熔断、进程内共享 3 QPS 限速、限并发分批、缓存水位增量、30 天前复权同源校准、原子缓存与来源状态。QPS 限制覆盖同一进程的全部腾讯 Provider 实例和重试请求，429 数值型 `Retry-After` 会延长退避。

数据对账见 [腾讯日线 Provider 数据对账报告](../reports/tencent-data-reconciliation.md)：54 只股票和 5 个指数的小窗口直连 59/59 成功，0 重试、0 失败；6 只股票的 2018 年至今全量基线为 54 请求、1.672 秒。AKShare 腾讯包装器对照确认北交所 8 个样本仍会触发 `KeyError: 'day'`，直连路径均成功。

验证结果：

```text

uv run ruff check src/     通过
uv run mypy src/           通过（52 个源文件）
```

残余风险：腾讯没有可引用的正式第三方财经数据字段文档或 SLA，同源包装器对账不能替代独立授权数据源验证；商业、公开或生产用途必须先完成授权和独立数据准确性复核，默认授权门禁会阻止误用。
